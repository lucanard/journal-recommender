# How to Run the Journal Recommender

## Prerequisites

- **Python 3.9+** — check with `python3 --version`
- **pip** — check with `pip --version`
- **~1 GB disk space** (for model + data)

That's it. No Docker, no database, no cloud services required.

---

## Option A: Fastest Path (5 minutes)

This gets you a working API with heuristic ranking (no API keys needed):

```bash
# 1. Go to the project folder
cd journal-recommender

# 2. Install dependencies
pip install fastapi uvicorn pydantic numpy sentence-transformers openpyxl

# 3. Generate embeddings (one-time, ~5 min)
cd data
python3 ../05_generate_embeddings.py --provider local
cd ..

# 4. Start the API
python3 app.py --data-dir data

# 5. Open in your browser
#    http://localhost:8000/docs
```

Test it with curl:
```bash
curl -X POST http://localhost:8000/recommend \
  -H "Content-Type: application/json" \
  -d '{
    "abstract": "We investigated the efficacy of a novel PD-1 inhibitor in patients with advanced non-small cell lung cancer through a randomized phase III trial.",
    "num_results": 3
  }'
```

## Option B: Best Quality (10 minutes, needs API key)

This adds Claude-powered explanations — much better recommendation quality:

```bash
# 1-3: Same as Option A above

# 4. Start with Claude re-ranking
python3 app.py --data-dir data --llm anthropic --llm-key YOUR_ANTHROPIC_API_KEY

# Or use environment variable instead:
export ANTHROPIC_API_KEY=sk-ant-your-key-here
python3 app.py --data-dir data --llm anthropic
```

```bash
cd journal-recommender

```

## Option D: Enrich First, Then Run (best data, ~3 hours)

For the highest quality recommendations, enrich the database before generating embeddings:

```bash
# 1. Install deps
pip install fastapi uvicorn pydantic numpy sentence-transformers openpyxl

# 2. Run enrichment pipeline (fetches metadata from 3 free APIs)
cd enrichment
cp journal_database.json .

#    Optional: edit your email in scripts 02 and 03 for 10x faster API access
#    (CONTACT_EMAIL variable near the top of each file)

python3 run_enrichment.py
#    This takes ~2-3 hours but saves progress every few minutes.
#    You can Ctrl+C and resume anytime.

# 3. Copy enriched data to the backend
cp journal_database_final.json ../data/
cp journal_embeddings_input.jsonl ../data/
cd ..

# 4. Generate embeddings from enriched data
cd data
python3 ../05_generate_embeddings.py --provider local
cd ..

# 5. Start
python3 app.py --data-dir data --llm anthropic --llm-key YOUR_KEY
```

## Connecting the Frontend

### Option 1: Use the React artifact in Claude.ai
The `frontend.jsx` file is a React component. It's already configured to call `http://localhost:8000`. Just render it in any React environment or use it directly as a Claude artifact.

### Option 2: Quick HTML page
Create an `index.html` that loads React from CDN and renders the component. Or use the API docs at `http://localhost:8000/docs` — FastAPI generates an interactive UI automatically.

### Option 3: curl / Postman / any HTTP client
The API is standard REST. The Swagger docs at `/docs` let you test every endpoint interactively in the browser.

## Troubleshooting

**"No module named 'fastapi'"**
→ Run: `pip install fastapi uvicorn pydantic numpy`

**"No module named 'sentence_transformers'"**
→ Run: `pip install sentence-transformers`
→ Or skip local embeddings: use `--provider openai` with an API key instead

**"No journal data found"**
→ Make sure `journal_database.json` (or `journal_database_final.json`) is in the `data/` folder

**"No embeddings found — Running in DEGRADED MODE"**
→ This is fine! The API still works, but it needs an LLM key to make recommendations.
→ To enable vector search, run `05_generate_embeddings.py` first.

**Slow embedding generation**
→ First run downloads the model (~90MB). Subsequent runs are faster.
→ Use `--provider openai` for ~30 second generation instead of ~5 minutes.

**Port 8000 already in use**
→ Run: `python3 app.py --data-dir data --port 8001`

## API Endpoints at a Glance

| Method | Path               | Purpose                          |
|--------|--------------------|----------------------------------|
| POST   | `/recommend`       | Get journal recommendations      |
| POST   | `/feedback`        | Submit feedback on a result      |
| GET    | `/health`          | System status & journal count    |
| GET    | `/stats`           | Detailed database statistics     |
| GET    | `/journals/search` | Search journals by name          |
| GET    | `/journals/{id}`   | Full details for one journal     |
| GET    | `/docs`            | Interactive API documentation    |
